10:00 – 12:00 SECTION 4: SHIELD MODE — ZK-SNARK PRIVACY

STORYBOARD: Navigate to /shield or use OmniTerminal in Shield mode. Walk through 4-step pipeline: Poseidon → Nullifier → PLONK → On-Chain Verify. Show loading animation during proof gen. Show success with tx hash.
Camera: Full browser, zoom to proof generation for dramatic effect.
Navigate to agt.finance/shield or toggle Shield Mode ON. Enter a shielded transaction.
Shield Mode uses real ZK-SNARK PLONK proofs — emphasis on "real." Most projects claiming ZK features use simulated proofs. We use production-grade cryptography.

Four-step cryptographic pipeline:

Step 1: Poseidon Hashing. Recipient, amount, and random secret fed into Poseidon — designed for ZK circuits, algebraically friendly, compact proofs. Our singleton Poseidon cache reduces proof time by 40% on repeated operations.

Step 2: Nullifier Generation. Unique nullifier prevents double-spending. Once submitted on-chain, it can never be reused. Reveals nothing about the original transaction.

Step 3: PLONK Proof Generation. Circom V2 circuit generates a PLONK proof using snarkjs — mathematical guarantee the transaction is valid without revealing underlying data. Takes 2-5 seconds.

Step 4: On-Chain Verification. Proof submitted to PlonkVerifierV2 (0x9FB9). Pass or fail, zero ambiguity. If pass, ShieldVaultV2 (0x3B4b) executes the transfer. Private, on-chain, cryptographically verified.
Show the proof generation loading animation. Let it play without speaking for dramatic effect.
